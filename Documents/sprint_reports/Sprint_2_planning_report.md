# SPRINT 2 - PLAN
## 29.1.2026 - 12.2.2026

### SPRINT GOAL
Main goal of the sprint is to start the development for the Attendance Checker-app.
This includes designing and starting the implementation of the UI for the Attendance Checker-app as well as developing and creating the relational database for the app. 
The plan is to also start writing some code functions and unit tests for them.

### SELECTED PRODUCT BACKLOG ITEMS
Here is the list of items that are selected to be implemented in this sprint:
- "Teacher can log in to app and log out of the app."
- "Teacher can view all their courses/classes."
- "Teacher can create new courses/classes."
- "Teacher can archive courses/classes."

### PLANNED TASKS
Here is the list of planned tasks for this sprint:
- Design the database --> Create ER-diagram and relational database diagram
- Implement the database --> Create the database creation script and needed seed data (students, teachers)
- Testing the database with CRUD methods --> Test that database works correctly (deletions only delete what they should etc.)
- Design the UI (with Figma) --> Wireframe for different views of the app
- Setup the project coding environment --> File architecture model (MVC?)
- Start implementing the UI in to the app --> Write code using JavaFX. Follow the Figma wireframe
- Start writing the code for the app --> See the selected product backlog items above
- Write unit tests for the code (functions) --> Test that the functions work with different inputs
- Integrate code coverage reporting --> Generate reports based on the unit tests

### TEAM CAPACITY & ASSUMPTIONS

### DEFINITION OF DONE
Sprint is done when the deadline is reached and all the tasks are finished. 
All the selected product backlog items <b>do not</b> need to be implemented for the sprint to be done. They will just be postponed for the next sprint.
