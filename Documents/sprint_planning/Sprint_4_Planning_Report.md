# SPRINT 4 - PLAN
## 5.3.2026 - 19.3.2026

### SPRINT GOAL
The main goal for this sprint is to finalize the functional prototype of the Attendance Checker app to a point where it can be presented to the class and shared for testing. 
The project should be possible to containerize with Docker and deployed in a suitable environment.

### SELECTED PRODUCT BACKLOG ITEMS

- "Teacher can create a new course"
- "Teacher can archive courses"

### PLANNED TASKS
Here is the list of planned tasks for this sprint:
- Implementing missing functionalities
- End-to-End testing --> All functionalities are implemented correctly, there are no major issues
- Cleaning the GUI --> Fix any visual issues
- Implementing Docker correctly:
  - Needs to work with JavaFX GUI --> Maven JavaFX Shade plugin
  - Containerization
  - Create image and push it to the Docker Hub
  - Image can be deployed correctly --> Test this
- Finalizing the project:
  - Updated documentation in GitHub and Trello
  - Completed README info and instructions explaining the project and its setup process
- Creating a presentation to share and show the project to other groups on the course

### TEAM CAPACITY & ASSUMPTIONS
Project has three members and the sprint is two weeks long. 
During those two weeks assuming that two days a week can be reserved for full-time project work, each team member can work about six hours a day so 12 hours a week and 24 hours total during the sprint. 
That makes the whole team capacity 72 hours for the whole sprint. 
These are just assumptions, the correct capacity calculation might differ based on the amount of tasks and other obligations outside the project.

### DEFINITION OF DONE
Sprint is done when the deadline is reached and all the tasks are finished and selected product backlog items are implemented. 
All the selected product backlog items <b>must be</b> implemented in this sprint. They cannot be postponed anymore.
