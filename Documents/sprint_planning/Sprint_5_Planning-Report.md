# SPRINT 5 - PLAN
## 24.3.2026 - 7.4.2026

### SPRINT GOAL
The main goal for this sprint is to prepare and implement full multilingual support for the application by localizing the user interface, planning and implementing GUI localization to enable future language scalability.

### SELECTED PRODUCT BACKLOG ITEMS
Here is the list of items that are yet to be implemented and must be implemented in this sprint:
- "Preparing for UI localization"
- "Implementing full GUI localization"

### PLANNED TASKS
Here is the list of planned tasks for this sprint:
- Update product backlog
- Choose languages for localization
- Ensure UI is ready for localization
- Identify and document all text elements, labels and messages requiring translation
- Language selection functionality in the UI
- GUI supports at least two non-Latin languages
- Document localization framework
- Update repository and documentation

### TEAM CAPACITY & ASSUMPTIONS
Project has three members and the sprint is two weeks long. 
During those two weeks assuming that two days a week can be reserved for full-time project work, each team member can work about six hours a day so 12 hours a week and 24 hours total during the sprint. 
That makes the whole team capacity 72 hours for the whole sprint. 
These are just assumptions, the correct capacity calculation might differ based on the amount of tasks and other obligations outside the project.

### DEFINITION OF DONE
Sprint is done when the deadline is reached and all the tasks are finished. All the selected product backlog items do not need to be implemented for the sprint to be done. 
They will just be postponed for the next sprint.
