# SPRINT 2 REVIEW REPORT
## 5.3.2026 - 17.3.2026

### Sprint Goal
Main goal of the sprint was to finalize the functional prototype of the Attendance Checker app. 
The project needs to be possible to containerize with Docker and deployed in a suitable environment.

### Completed User Stories / Tasks
The final two user stories were completed:

- "As a teacher, I want to create new courses/classes, so that I can create and keep track of the attendance of different student compositions and courses efficiently."
- "As a teacher, I want to archive completed courses/classes, so that I can keep the active courses/classes list clean and simple."

Other completed tasks are:

- End-to-End testing and bug fixes
- Basic clean up for the GUI
- Implementing Docker to work with JavaFX GUI
- Docker containerization, image creation and pushing the image to Docket Hub
- Updating the documentation in GitHub and Trello
- Updating the README with info and instructions explaining the project and its setup process
- Creating a presentation to share and show the project to other groups on the course

### Demo Summary
There was no actual demo for this sprint, but the project was presented to the product owner (teacher) and the other groups on the course. The following items were presented:

- Project and sprint overviews
- Use Case diagram, ER diagram and realtional schema
- Technology stack
- Attendance Checker application demo
- Running the app in a Docker container
- Product validation and realiability (i.e. testing)
- Future development plans

### What Went Well
All tasks goals and tasks that were planned for this sprint were reached and done.
The product works and includes all the planned functionalities.
It is possible to containerize the project and deploy it. The app also works correctly when run through a container.
The CI/CD Pipeline is fully implemented.

### What Could Be Improved
Group's teamwork is still an issue as work is still being distributed unevenly. 
Code needs documentation/comments for improved communication and understanding.

### Next Sprint Focus
The focus of the next sprint is to learn about localization and through that localize the GUI of the app with two different non-Latin languages.
